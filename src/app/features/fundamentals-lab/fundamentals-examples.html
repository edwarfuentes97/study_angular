🎯 Directiva Custom: appHighlight

Pasa el mouse sobre los elementos para ver la directiva en acción. Cambia el color con el picker.

{{ highlightColor }}

🟢 Hover para highlight con color custom

🔵 Highlight azul fijo

🟠 Highlight naranja fijo

⚪ Highlight default (#58a6ff)

Código de la directiva:

@Directive({ selector: '[appHighlight]' })
export class HighlightDirective {
  @Input() appHighlight = '#58a6ff';
  private el = inject(ElementRef);

  @HostListener('mouseenter') onMouseEnter() {
    this.el.nativeElement.style.backgroundColor = this.appHighlight;
  }
  @HostListener('mouseleave') onMouseLeave() {
    this.el.nativeElement.style.backgroundColor = '';
  }
}

Custom Pure Pipe: filterBy

@for (fruit of fruits | filterBy:'name':searchTerm | sortBy:'name'; track fruit.name) {
{{ fruit.name }} {{ fruit.category }}
} @empty {

No se encontraron resultados para "{{ searchTerm }}"

}
💡 Pure vs Impure: filterBy es pure — solo se re-ejecuta si searchTerm cambia por referencia. sortBy es impure (pure: false) — se re-ejecuta en cada ciclo de Change Detection, lo cual es necesario para que funcione con pipes encadenados pero tiene costo de rendimiento.

🎨 ViewEncapsulation — Comparación

Emulated (Default)

Este texto tiene estilos encapsulados

_nghost-abc y _ngcontent-abc
Angular añade atributos únicos al HTML y al CSS.
CSS: p[_ngcontent-abc] { color: red; }

ShadowDom (Nativo)

Este texto usa Shadow DOM real

Usa #shadow-root del navegador.
CSS completamente aislado — no entra ni sale.
Limitación: No todos los componentes/librerías funcionan bien.

None (Global)

¡Estos estilos son GLOBALES!

Sin encapsulación. CSS se aplica a toda la app.
Peligro: Puede romper estilos de otros componentes.
Uso: Estilos base de librería, reset CSS.

Cómo se configura:

@Component({
  encapsulation: ViewEncapsulation.Emulated  // default — no necesitas declarar
  encapsulation: ViewEncapsulation.ShadowDom // Shadow DOM nativo
  encapsulation: ViewEncapsulation.None      // sin encapsulación
})
🎓 En entrevistas: "¿Cuál es la diferencia entre Emulated y ShadowDom?" — Emulated usa atributos HTML para simular aislamiento; ShadowDom usa la API nativa del navegador. Emulated es el default porque funciona en todos los escenarios.
