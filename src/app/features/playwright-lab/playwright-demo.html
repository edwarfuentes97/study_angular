Configuración de ejemplo

Usa un archivo de configuración para definir directorio, browsers y servidor. Esto garantiza que Angular esté disponible antes de ejecutar pruebas.

Page Object Model

Centraliza selectores y acciones en clases para que los tests se mantengan legibles.

Fixture personalizado

Crea un fixture que navegue a la página de inicio y devuelva el objeto de página para pruebas.

Ejemplo de prueba

Con un fixture y un page object la prueba queda muy sencilla y fácil de leer.

Elementos reutilizables

Encapsula inputs, botones y componentes compartidos en clases de elemento para facilitar la reutilización.

Estado de autenticación

Almacenar el estado de la sesión en un archivo permite iniciar tests autenticados sin repetir logins. Esto mejora la velocidad y mantiene los tests independientes.
