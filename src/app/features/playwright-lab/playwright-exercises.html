Ejercicio 1: Configura Playwright

Ejecuta npm init playwright@latest y elige TypeScript como lenguaje. Crea la configuración básica y verifica que los navegadores estén instalados.

Asegúrate de definir webServer para que Playwright levante tu app Angular.

Ejercicio 2: Agrega selectores test-id

Añade data-testid a elementos críticos como el título de inicio, campos de filtro y botones. Estos selectores mejoran la estabilidad de los tests frente a cambios de estilo.

Ejercicio 3: Crea un Page Object

Implementa una clase StartPage con método goto() y selectores para los elementos clave. Usa esta clase en tus pruebas para separar navegación de assertions.

Ejercicio 4: Escribe una prueba simple

Escribe un test que abra la página de inicio y verifique el mensaje de bienvenida. Usa expect(...).toContainText(...) para validar el contenido.

Ejercicio 5: Reutiliza un elemento compartido

Crea una clase FilterInputElement que encapsule el campo de filtro y el botón de reset. Luego úsala desde un page object para interactuar con el formulario.

Ejercicio 6: Prueba el modo UI y debug

Ejecuta npx playwright test --ui y explora la interfaz. Luego ejecuta npx playwright test --debug para inspeccionar los pasos y usar el inspector.

Ejercicio 7: Agrega un flujo autenticado

Crea un setup de autenticación que guarde el estado en un archivo y reutilízalo en pruebas posteriores. Esto te ayudará a testear rutas protegidas sin repetir el login.
