1. Change Detection

Angular usa Zone.js para interceptar eventos asíncronos (clicks, HTTP, timers) y disparar un ciclo de Change Detection (CD) desde la raíz del árbol de componentes. En modo Default, cada componente se revisa en cada ciclo. Con OnPush, Angular solo revisa el componente cuando cambian sus @Input() por referencia, se dispara un evento del template, o se llama manualmente a markForCheck(). Esto reduce drásticamente el trabajo del CD en apps grandes. 

// Componente con OnPush — solo se re-renderiza si cambia la referencia del input
@Component({
  selector: 'app-child',
  imports: [AutoGlossaryDirective],
  changeDetection: ChangeDetectionStrategy.OnPush,
  template: \`<p>{{ data.name }}</p>\`
})
export class ChildComponent {
  @Input() data!: { name: string };
}

// En el padre — mutar NO dispara CD en OnPush
this.user.name = 'Nuevo';           // ❌ misma referencia
this.user = { name: 'Nuevo' };     // ✅ nueva referencia

2. Signals

Signals son primitivas reactivas introducidas en Angular 16+ que permiten rastrear estado de forma granular sin Zone.js. signal() crea un valor reactivo, computed() deriva valores automáticamente, y effect() ejecuta side-effects cuando las dependencias cambian. linkedSignal() (Angular 19+) conecta un signal derivado que también puede escribirse. Comparado con RxJS, los Signals son sincrónicos, más simples para estado local, y se integran con templates sin async pipe. 

import { signal, computed, effect, linkedSignal } from '@angular/core';
import { toSignal } from '@angular/core/rxjs-interop';

// signal — valor reactivo mutable
const count = signal(0);
count.set(5);
count.update(v => v + 1);

// computed — derivado, se recalcula automáticamente
const double = computed(() => count() * 2);

// effect — side-effect que corre cuando cambian sus deps
effect(() => console.log('Count:', count()));

// linkedSignal — signal derivado que también es writable
const selected = linkedSignal(() => items()[0]);
selected.set(items()[2]); // se puede sobrescribir manualmente

// Integración con RxJS
const data = toSignal(this.http.get('/api'), { initialValue: [] });

3. Reactive Forms

Reactive Forms ofrecen un modelo imperativo para construir formularios complejos. FormGroup agrupa controles, FormControl maneja valores individuales, y FormArray permite listas dinámicas. Los Validators built-in cubren casos comunes (required, minLength, pattern), y puedes crear validadores custom como funciones puras. El estado del formulario (valid, dirty, touched) se actualiza automáticamente. 

import { FormGroup, FormControl, FormArray, Validators } from '@angular/forms';

// FormGroup con validadores
const form = new FormGroup({
  name:  new FormControl('', [Validators.required, Validators.minLength(3)]),
  email: new FormControl('', [Validators.required, Validators.email]),
  skills: new FormArray([new FormControl('Angular')])
});

// Agregar dinámicamente al FormArray
(form.get('skills') as FormArray).push(new FormControl('RxJS'));

// Validador custom
function forbiddenName(name: string): ValidatorFn {
  return (control: AbstractControl) =>
    control.value === name ? { forbiddenName: { value: control.value } } : null;
}

// Acceder a errores
form.get('email')?.errors;    // { required: true } | { email: true } | null
form.get('email')?.touched;   // boolean

4. Lazy Loading

Lazy loading permite cargar módulos o componentes standalone bajo demanda, reduciendo el bundle inicial. Con loadComponent se carga un componente standalone directamente en una ruta. Con loadChildren se carga un archivo de rutas completo. Las estrategias de preloading (PreloadAllModules, NoPreloading, o custom) controlan cuándo se descargan los chunks en segundo plano. 

import { Routes } from '@angular/router';
import { PreloadAllModules } from '@angular/router';

// Lazy load de un componente standalone
export const routes: Routes = [
  {
    path: 'dashboard',
    loadComponent: () =>
      import('./dashboard/dashboard').then(m => m.DashboardComponent)
  },
  // Lazy load de rutas hijas
  {
    path: 'admin',
    loadChildren: () =>
      import('./admin/admin.routes').then(m => m.ADMIN_ROUTES)
  },
];

// Preloading strategy en provideRouter
provideRouter(routes, withPreloading(PreloadAllModules));

5. Interceptors & Guards

Desde Angular 15+, los interceptors y guards se escriben como funciones en vez de clases. Los interceptors funcionales reciben req y next, permitiendo modificar requests, manejar errores, y encadenar lógica. Los guards funcionales (CanActivateFn, CanMatchFn) simplifican la protección de rutas usando inject() para acceder a servicios. Se registran con withInterceptors() en provideHttpClient(). 

import { HttpInterceptorFn } from '@angular/common/http';
import { inject } from '@angular/core';
import { CanActivateFn, Router } from '@angular/router';

// Interceptor funcional — agrega token y maneja 401
export const authInterceptor: HttpInterceptorFn = (req, next) => {
  const token = inject(AuthService).getToken();
  const cloned = req.clone({
    setHeaders: { Authorization: \`Bearer ${token}\` }
  });
  return next(cloned).pipe(
    catchError(err => {
      if (err.status === 401) inject(Router).navigate(['/login']);
      return throwError(() => err);
    })
  );
};

// Guard funcional
export const authGuard: CanActivateFn = () =>
  inject(AuthService).isLoggedIn() || inject(Router).createUrlTree(['/login']);

// Registro
provideHttpClient(withInterceptors([authInterceptor]));
